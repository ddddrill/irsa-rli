## О проекте

**IRSA** (Inverse-synthetic-aperture Radar Simulator & Analyzer) — это программный симулятор, который моделирует работу **радара с инверсной синтезированной апертурой (ISAR)** и формирует радиолокационные изображения (РЛИ) космических аппаратов на основе физически корректной модели движения.

Проект показывает **полный цикл обработки радиолокационного сигнала** — от сырых отсчётов АЦП до финального изображения, на котором видна форма спутника. Каждый этап можно запустить, посмотреть промежуточные результаты и сравнить разные алгоритмы.

В отличие от учебных 2D-моделей, IRSA учитывает:
- честную кинематику цели (поступательное + вращательное движение),
- реальную геометрию съёмки (SFCW-радар с конечной полосой и углом обзора),
- необходимость компенсации поступательного движения (MOCOMP),
- эффекты больших углов обзора (полярная переформатировка).

- **Интеграционный тест** с автопроверками сфокусированности РЛИ

## Физика: как радар «видит» спутник

### Что такое ISAR

Обычный радар измеряет **дальность** до объекта — расстояние по лучу. Этого мало: непонятно, **где** в поперечном направлении (по азимуту) находится отражатель.

**ISAR (Inverse Synthetic Aperture Radar)** решает эту задачу за счёт **вращения цели**. Если объект поворачивается относительно радара, разные точки на нём имеют разную лучевую скорость → разный доплеровский сдвиг → их можно разделить по частоте с помощью БПФ.

> Представьте вертушку: лопасти вращаются, кончик каждой лопасти движется быстрее основания. Радар это «видит» как разные доплеровские частоты.

### Сигнал SFCW

В проекте используется **SFCW (Stepped Frequency Continuous Wave)** — радар, который излучает N импульсов на последовательно меняющихся частотах:

```
Частота:   f₀,  f₀+Δf, f₀+2Δf, ..., f₀+(M-1)·Δf
Импульс:   n=0,  n=1,  n=2,   ...,  n=N-1
```

Каждый отсчёт — это комплексное число (I/Q), пропорциональное напряжённости поля, отражённого целью. На выходе формируется **матрица E[M×N]** (частоты × импульсы).

### Дальность и азимут

- **Дальность (Range)** восстанавливается по **быстрому времени** (внутри одного импульса) через IFFT по оси частот. Разрешение: $\Delta r = c / (2B)$, где $B$ — полоса сигнала.
- **Азимут (Cross-Range)** восстанавливается по **медленному времени** (между импульсами) через FFT по оси доплеровских частот. Разрешение: $\Delta x = \lambda / (2 \omega T_{\text{obs}})$, где $\omega$ — угловая скорость вращения цели.

### Почему нужно учитывать движение

В реальности спутник **не просто вращается** — он ещё и **движется по орбите** со скоростью ~7.5 км/с. Это поступательное движение приводит к:
- **Range Walk** — каждая точка «ползёт» по бинам дальности от импульса к импульсу (косые линии на профилях).
- **Фазовому набегу** — линейный фазовый тренд по медленному времени, размывающий азимутальное FFT.

Без компенсации РЛИ превращается в кашу. **MOCOMP** (Motion Compensation) — обязательный шаг перед азимутальным сжатием.

### Проблема больших углов

При малых углах обзора ($\Delta\theta < 3°$) данные в **k-пространстве** (пространстве волновых чисел) лежат почти на прямоугольной сетке, и 2D БПФ работает корректно. Но при больших углах (типично для ISAR X-диапазона на длинных экспозициях) сетка становится **кольцевым сектором** — и 2D БПФ даёт **arc smearing** (размазню в виде дуг).

Решение — **полярное переформатирование**: пересчёт данных с полярной сетки на декартову перед 2D IFFT.

## Реализованные алгоритмы

### 1. Генерация сырых данных (`simulation/raw_generator.py`)

**Цель:** сформировать комплексную матрицу E[M×N] эхо-сигнала SFCW по известной геометрии цели.

**Физика:** для каждого точечного рассеивателя цели вычисляется
$$E[m,n] = \sum_i \sigma_i \cdot \exp\left(-j\frac{4\pi f_m R_i(t_n)}{c}\right),$$
где $R_i(t_n)$ — мгновенная дальность до $i$-й точки в момент $n$-го импульса, $\sigma_i$ — её ЭПР.

**Назначение в проекте:** даёт детерминированный, повторяемый вход для всех остальных алгоритмов — не нужен реальный радар.

### 2. Сжатие по дальности (`processing/range_compress.py`)

**Цель:** преобразовать «сырую» матрицу частотных отсчётов в **профили дальности** — по сути, восстановить, на каких дальностях находятся отражатели.

**Алгоритм:** 1D IFFT по оси частот + дешифровка (dechirp — переход от дальности относительно опорной $R_0$ к абсолютной дальности).

**Назначение:** превращает частотную ось в дальностную. Без этого шага невозможно сказать, где находится цель.

### 3. MOCOMP (`processing/mocomp.py`)

**Цель:** убрать вредное влияние поступательного движения на сигнал.

**Два шага:**
1. **Выравнивание огибающей** (Envelope Alignment, раздел 8.3.1 Özdemir) — кросс-корреляция соседних профилей, оценка и устранение Range Walk.
2. **Коррекция фазы** (Phase Autofocus, раздел 8.3.3 Özdemir) — выделение опорного бина (самого сильного отражателя) и устранение линейного фазового тренда.

**Назначение:** без MOCOMP РЛИ при $V \neq 0$ полностью размыто. Это критический этап для любой реальной цели.

### 4. Стандартное азимутальное сжатие (`processing/azimuth_compress.py`)

**Цель:** разделить отражатели по азимуту (cross-range).

**Алгоритм:** 1D FFT по оси медленного времени (импульсов) → переход от доплеровской частоты к физическому азимуту через $\Delta x = \lambda / (2\omega T_{\text{obs}})$. Опционально — взвешивание оконной функцией (hamming, hann) для подавления боковых лепестков.

**Ограничение:** работает корректно **только при малых углах** ($\Delta\theta < 3°$). На больших углах — arc smearing.

### 5. Полярное переформатирование (`processing/polar_reformat.py`)

**Цель:** снять ограничение на угол обзора, дать чёткое РЛИ при $\Delta\theta \geq 3°$.

**Физическая суть:** данные SFCW лежат на полярной сетке в k-пространстве $(k, \theta)$, а 2D БПФ требует декартовой $(k_x, k_y)$. При больших углах полярная сетка превращается в «кусок пончика» — и 2D FFT на нём даёт дуги.

**Алгоритм (6 шагов):**
1. **Полярная сетка:** $k_m = 4\pi f_m / c$, $\theta_n = \omega n \cdot \text{PRI}$ → кольцевой сектор в k-space
2. **Декартова сетка:** $\Delta k_x = 2\pi / X_{\max}$, $\Delta k_y = 2\pi / Y_{\max}$ по теореме Котельникова
3. **Билинейная интерполяция** полярная → декартова (scipy RegularGridInterpolator)
4. **Окно Тейлора** (nbar=4, SLL=−35 dB) — подавление боковых лепестков от резких краёв сектора
5. **Zero-padding ×2** — визуальная sinc-интерполяция без потери информации
6. **2D IFFT** + физическое масштабирование осей в метры

**Метрики качества:**
- **Энтропия изображения** `compute_image_entropy` — мера размазанности. Ниже = лучше фокусировка.
- **ISNR** `compute_isnr` — отношение сигнал/шум. Выше = лучше.

**Автоматическое переключение:** `should_use_polar_reformat(omega, pri, N, threshold_deg=3.0)` — решает, какой конвейер использовать, по углу $\Delta\theta = \omega \cdot N \cdot \text{PRI}$.

## Конвейер обработки

**Стандартный (малые углы, Δθ < 3°):**

```
┌─────────────┐   ┌─────────────────┐   ┌────────┐   ┌─────────────────────┐   ┌──────────────┐
│  SFCW-      │   │  Сжатие по      │   │        │   │  Сжатие по          │   │  Финальное   │
│  генерация  │ → │  дальности      │ → │  MOCOMP │ → │  азимуту            │ → │  РЛИ         │
│  E[M×N]     │   │  (1D IFFT)      │   │        │   │  (1D FFT)           │   │  |I|[M×N]    │
│             │   │  P[M×N]         │   │ P_comp │   │  I[M×N]             │   │              │
└─────────────┘   └─────────────────┘   └────────┘   └─────────────────────┘   └──────────────┘
```

**Полярное переформатирование (большие углы, Δθ ≥ 3°):**

```
┌─────────────┐   ┌─────────────────┐   ┌────────┐   ┌─────────────────────┐   ┌──────────────┐
│  SFCW-      │   │  Сжатие по      │   │        │   │  Полярное           │   │  Финальное   │
│  генерация  │ → │  дальности      │ → │  MOCOMP │ → │  переформатирование │ → │  РЛИ         │
│  E[M×N]     │   │  (1D IFFT)      │   │        │   │  + 2D IFFT          │   │  |I|[M×N]    │
│             │   │  P[M×N]         │   │ P_comp │   │  (k,θ)→(kx,ky)      │   │              │
└─────────────┘   └─────────────────┘   └────────┘   └─────────────────────┘   └──────────────┘
                                                       ↓
                                          билинейная интерполяция
                                          + окно Тейлора + zero-pad
```

| Этап | Модуль | Что делает |
|------|--------|------------|
| Генерация | `simulation/raw_generator.py` | E[m,n] = Σ σᵢ · exp(−j4πfₘRᵢ(tₙ)/c) |
| Сжатие по дальности | `processing/range_compress.py` | 1D IFFT по частотам + дешифровка |
| MOCOMP | `processing/mocomp.py` | Envelope alignment + phase autofocus |
| Сжатие по азимуту (стандарт) | `processing/azimuth_compress.py` | 1D FFT по импульсам |
| **Полярное переформатирование** | `processing/polar_reformat.py` | (k,θ) → (kx,ky), билин. интерполяция, Тейлор, 2D IFFT |

## Когда нужна полярная переформатировка

Правило: если суммарный угол поворота цели за время наблюдения превышает **3°**, используйте полярный метод.

```python
Δθ = ω · N · PRI
```

Типичные случаи:
- **X-диапазон** (8–12 ГГц), большие спутники: Δθ быстро набегает, полярка нужна почти всегда
- **L-диапазон** (1–2 ГГц): Δθ набегает медленнее, стандартного метода хватает дольше
- **Длинные экспозиции** (много импульсов при высоком PRI): даже малая ω даёт большое Δθ

**Проверить порог в коде:**
```python
from processing.polar_reformat import should_use_polar_reformat
use_polar, dtheta_rad, dtheta_deg = should_use_polar_reformat(omega, pri, N, threshold_deg=3.0)
print(f"Δθ = {dtheta_deg:.2f}°  →  {'POLAR' if use_polar else 'STANDARD'}")
```

## Графический интерфейс

PyQt5 + pyqtgraph. Окно разделено на две части:

**Верх** — ввод параметров (3 секции через горизонтальный QSplitter):
- **Исследуемый спутник** — выбор КА и краткая справка
- **Параметры** — радар, обзор, кинематика, обработка
- **Управление** — кнопки «Показать КА», «Рассчитать РЛИ», «Запустить поток»

**Низ** — 5 вкладок с результатами:

| Вкладка | Что показывает |
|---------|----------------|
| **Сигнал** | Параметры SFCW-импульса, лесенка во временной области, спектр |
| **Профили дальности** | Heatmap профилей дальности после IFFT (дальность × импульсы) |
| **Компенсация движения** | Сдвиги огибающей, фазовые ошибки, heatmaps до/после MOCOMP |
| **РЛИ** | Финальное ISAR-изображение (X × Y в метрах) |
| **Поток кадров** | Анимация покадрового расчёта |

**Параметры ввода:**
- Выбор спутника (5 КА) и метода обработки (стандартный / с полярным переформатированием)
- Радар: f_c, B, nifft_size, beam_width, range
- Обзор: Xmax, Ymax (размер сцены)
- Кинематика: V, α, ω, PRI, N
- Обработка: MOCOMP вкл/выкл, оконная функция, режим отображения linear/dB
- Ко всем полям — tooltips с физическим смыслом и формулами

## Параметры по умолчанию

| Параметр | Значение | Физический смысл |
|----------|----------|------------------|
| `method` | стандартный | Метод обработки (стандартный / с полярным переформатированием) |
| `f_c` | 8.0 ГГц | Центральная частота (X-диапазон) |
| `B` | 0.5 ГГц | Полоса сигнала → Δr = c/(2B) = 0.3 м |
| `nifft_size` | 1024 | Размер IFFT (степень 2) |
| `beam_width` | 9.0° | Ширина ДН антенны |
| `R₀` | 500 км | Начальная дальность до объекта |
| `Xmax` | 20 м | Размер сцены по дальности |
| `Ymax` | 20 м | Размер сцены по азимуту |
| `V` | 0 м/с | Поступательная скорость (0 = turntable) |
| `α` | 0° | Угол направления движения |
| `ω` | 0 (auto) | Угловая скорость (авто из beam_width) |
| `PRI` | 1.0 с | Период повторения импульсов |
| `N_pulses` | 1000 | Количество импульсов (= размер оси азимута) |
| `MOCOMP` | вкл | Компенсация движения |
| `window` | none | Оконная функция (none / hamming / hann) |
| `display_mode` | dB | Режим отображения (linear / dB) |

**Связь параметров с качеством РЛИ:**

| Что хотим | Что крутить |
|---|---|
| Лучше разрешение по дальности | ↑ B (полоса), ↑ nifft_size |
| Лучше разрешение по азимуту | ↑ N, ↑ f_c, ↓ ω (узкая ДН) |
| Меньше боковых лепестков | Включить window (hamming/hann) |
| Чётче при больших Δθ | Включить полярное переформатирование |
| Подавить поступательное движение | Включить MOCOMP |

## Тестирование

Перед отправкой изменений проверь:

1. **GUI запускается:** `run.bat` или `.build_venv\Scripts\python.exe main.py`
2. **Все 5 вкладок заполняются** при нажатии «Рассчитать РЛИ»
3. **Метод «с полярным переформатированием»** даёт сфокусированное изображение
4. **Все 6 демо-скриптов** запускаются без ошибок и сохраняют PNG

### Что считать «поломанным»

| Ситуация | Ожидание |
|----------|----------|
| Стандартный метод, Δθ < 3°, V=0 | Точки сфокусированы |
| Стандартный метод, Δθ < 3°, V≠0 + MOCOMP | Точки сфокусированы (Range Walk и фаза скомпенсированы) |
| Стандартный метод, Δθ ≥ 10° | Ожидаемы дуги/размазня (arc smearing) |
| Полярный метод, Δθ ≥ 3° | Точки сфокусированы |
| Полярный метод, Δθ ≥ 3°, V≠0 + MOCOMP | Точки сфокусированы |
| dB-режим | Пиксели фона −30…−50 dB, точки около 0 dB |
| linear-режим | Большинство значений близко к 0, яркие пятна — единицы (выглядит «тёмным») |
| `energy in top-1%` > 30% | Изображение сфокусировано |
